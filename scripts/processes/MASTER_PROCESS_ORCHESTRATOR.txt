## TEMPLATE INFORMATION
- **Purpose**: Coordinates master picture analysis process with version control and automated classification
- **Use Case**: Main entry point for any picture analysis workflow
- **Processing Time**: Variable (depends on sub-components)
- **Output Format**: Coordinated multi-component execution with status reporting

## CORE PRINCIPLE
**MODULAR ORCHESTRATION**: Coordinates independent components while maintaining version control and process integrity across all sub-systems.

## EXECUTION METHODOLOGY

### 1. Process Initialization
```
=== MASTER PROCESS ORCHESTRATOR - EXECUTION LOG ===
Process Start Time: [TIMESTAMP]
Orchestrator Version: 1.0.0
Target Directory: [USER_SPECIFIED]
Components to Execute: VERSION_CONTROL_MANAGER, AUTOMATED_IMAGE_CLASSIFIER
```

### 2. Component Execution Sequence
```
STEP 1: VERSION_CONTROL_MANAGER
├─ Purpose: Apply version control standards to all outputs
├─ Input: Process metadata, target directory
├─ Output: Version-controlled headers and tracking
└─ Status: [SUCCESS/FAILURE]

STEP 2: AUTOMATED_IMAGE_CLASSIFIER  
├─ Purpose: Classify images and update CSV database
├─ Input: Image directory, classification criteria
├─ Output: Updated image_classifications.csv
└─ Status: [SUCCESS/FAILURE]
```

### 3. Integration Points
```
INTER-COMPONENT COMMUNICATION:
├─ VERSION_CONTROL_MANAGER → Provides version headers to all components
├─ AUTOMATED_IMAGE_CLASSIFIER → Uses version control for CSV updates
└─ ORCHESTRATOR → Monitors and reports status of all components
```

## ORCHESTRATION WORKFLOW

### Pre-Execution Validation
```
VALIDATION CHECKLIST:
□ Target directory exists and contains images
□ VERSION_CONTROL_STANDARDS.md accessible
□ IMAGE_CLASSIFICATION_DOCUMENTATION.txt accessible  
□ image_classifications.csv exists and writable
□ All component prompts available (v1.0.0)
```

### Execution Sequence
```
1. INITIALIZATION PHASE:
   ├─ Log process start with timestamp
   ├─ Validate all required files and directories
   ├─ Initialize component status tracking
   └─ Generate unique process execution ID

2. VERSION CONTROL PHASE:
   ├─ Execute: VERSION_CONTROL_MANAGER_v1.0.0
   ├─ Input: Process metadata, execution context
   ├─ Expected Output: Version headers for all subsequent outputs
   └─ Validation: Confirm version control standards applied

3. CLASSIFICATION PHASE:
   ├─ Execute: AUTOMATED_IMAGE_CLASSIFIER_v1.0.0
   ├─ Input: Target directory, version control headers
   ├─ Expected Output: Classifications appended to CSV
   └─ Validation: Confirm all images processed and classified

4. COMPLETION PHASE:
   ├─ Aggregate component results
   ├─ Generate execution summary
   ├─ Apply final version control to summary
   └─ Report overall process status
```

### Error Handling
```
ERROR MANAGEMENT:
├─ Component Failure: Log error, attempt recovery, continue with remaining components
├─ File Access Issues: Validate permissions, provide clear error messages
├─ Version Control Errors: Fallback to manual versioning with warnings
└─ Classification Errors: Log failed images, continue with successful ones
```

## COMPONENT INTEGRATION SPECIFICATIONS

### VERSION_CONTROL_MANAGER Interface
```
INPUT SPECIFICATION:
- process_context: Object containing execution metadata
- target_outputs: List of files/outputs requiring version control
- component_versions: Dictionary of component versions

OUTPUT SPECIFICATION:
- version_headers: Standardized headers for all outputs
- execution_metadata: Version-controlled process tracking
- compliance_status: Version control validation results
```

### AUTOMATED_IMAGE_CLASSIFIER Interface
```
INPUT SPECIFICATION:
- image_directory: Path to images for classification
- classification_criteria: Reference to IMAGE_CLASSIFICATION_DOCUMENTATION.txt
- csv_target: Path to image_classifications.csv
- version_control_data: Headers and metadata from VERSION_CONTROL_MANAGER

OUTPUT SPECIFICATION:
- classification_results: List of classified images with confidence scores
- csv_updates: Number of new entries added to CSV
- processing_errors: List of failed classifications with error details
```

3. ID: 2356796
- Contents: 6.1.1.1.1.1.1.3 Diagnosis and SW Updates implementation shall communicate with end-user applications to provide the following diagnosis functionality:

- Responsible Domain: IOC_Diag

4. ID: 2356797
- Contents: 6.1.1.1.1.1.1.4 Diagnosis and SW Updates implementation shall communicate with external services and devices to provide the following functionality:

 

OTA updates from cloud servers via "OTA SWP" and Communication vehicle device

local updates from media storage(s) via USB connection

CAN & Ethernet diagnosis interaction with connected OBD device
- Artifact Type: SRS Req
- Responsible Domain: IOC_Diag

5. ID: 2356798
- Contents: 6.1.1.1.1.1.1.5 Diagnosis and SW Updates implementation shall communicate with the platform component to provide the following functionality:

 

correct Power Management sequences during active operation, e.g., preventing sleep mode during SW update

correct firmware image flashing according to the A/B partitions rule and file system operation

communication via CAN and Ethernet channels according to UDS and DoIP protocols

cybersecurity management in the scope of key management and authentication (encrypt, decrypt, and signature services)

access to failure artifacts according to Heath management and Logging rules
- Artifact Type: SRS Req
- Responsible Domain: IOC_Diag

6. ID: 2356799
